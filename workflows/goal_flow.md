# Goal Flow

## 目的
ユーザーの依頼を、状態更新、計画、実行、検証、次アクションへ落とし込みます。

## 手順
1. `loops/task_state.md` を読む
2. 依頼が要件整理、設計、試作、実装、検証、改善のどれか判定する
3. 必要なSkillを選ぶ
4. 最小の実行単位を決める
5. 実行後に検証する
6. `loops/iteration_log.md` と `loops/task_state.md` を更新する

## /goal 候補
- `中学校時間割アプリの要件整理と最小試作計画を作成する`
- `サンプルデータ構造とハード制約チェック方針を設計する`
- `最小試作の技術スタック候補を比較する`
