# Project Spec

## 背景
中学校の時間割作成は、教員、クラス、教科、時限、特別教室、固定授業、非常勤勤務日など多数の条件を同時に扱う業務です。ユーザーは詳細仕様をまだ十分に提示できないため、Codexが要件と不足情報を整理しながら進めます。

## 最終目的
教務担当者、管理職、時間割作成担当者が使える、中学校向け時間割自動作成Webアプリを作ることです。

## 想定ユーザー
- 中学校の教務担当者
- 管理職
- 時間割作成担当者
- ICTに詳しくないが、表計算ソフト程度は使える学校関係者

## 対象業務
- 学校設定、教員、教科、教室、固定枠、制約の入力
- ユーザーが画面上で時間割作成に必要な情報を入力する
- 必須条件を守る時間割案の自動生成
- 複数の時間割候補を画面に表示し、違反や評価を比較する
- 条件違反、偏り、空き時間、教員重複、教室重複の検出
- 画面での確認と手修正
- CSVまたはJSONによる入出力
- 必要に応じた印刷、共有、改善

## 想定機能
- 学校基本設定
- 教員、教科、クラス、教室、固定枠の管理
- 画面入力による条件登録
- ハード制約とソフト制約の管理
- 時間割自動生成
- 複数候補の生成と候補比較
- 違反検出とスコアリング
- 手修正と部分固定
- サンプルデータ検証
- CSVまたはJSON入出力
- 後続候補として印刷用表示、PDF出力、保存履歴

## 技術方針
- 完全無料のローカルWebアプリとして開発する
- 初期試作はHTML/CSS/Vanilla JavaScriptの静的Webアプリを第一候補にする
- 有料API、外部API、外部CDNは使用しない
- 時間割生成、検証、保存はローカルブラウザ内で完結させる

## 入力データ
- 曜日、時限、学年、クラス数
- 教科ごとの週時数
- 教員の担当教科、担当クラス、勤務不可時間
- 教室種別と利用可能枠
- 固定授業、行事、会議
- 優先したい条件と避けたい条件
- 必ず守りたい条件と、できれば守りたい条件

## 出力データ
- クラス別時間割
- 教員別時間割
- 教室別利用表
- 候補ごとの評価スコア
- 制約違反一覧
- スコアと改善候補
- CSVまたはJSON形式のエクスポート

## ハード制約例
- 同じ教員が同じ時限に複数クラスを担当しない
- 同じ特別教室が同じ時限に複数授業へ割り当てられない
- 各クラスの各教科が必要時数を満たす
- 教員の勤務不可時間には授業を入れない
- 固定授業枠を変更しない
- クラスに同時刻で複数授業を入れない

## ソフト制約例
- 主要教科が特定曜日や午後に偏りすぎない
- 同じ教科が同じ日に重なりすぎない
- 教員の空き時間や連続授業をなるべく平準化する
- 特別教室利用をなるべく分散する
- 非常勤教員の出勤日をなるべく少なくする
- クラスごとの日々の負荷をなるべく均等にする

## 非対象範囲
- 初期段階での認証、複数校管理、高度な権限管理
- 未承認の外部サービス連携
- 有料APIやクラウドAPIに依存する機能
- 個人情報を含む実データの外部送信
- ハード制約違反を許した最終確定

## 初期仮定
詳細は `context/assumptions.md` に分離します。初期は、日本の一般的な中学校、週5日、1日6時限、1週間分の標準時間割を対象とします。

## 未確認事項
- 実際の学校規模
- 曜日、時限、短縮日課の扱い
- 教科ごとの正式な週時数
- 教員の担当範囲と勤務不可時間
- 特別教室と体育館の利用制約
- 固定授業、会議、行事の頻度

## 成功条件
- 必須情報と未確認事項が分離されている
- ハード制約とソフト制約が明確に分かれている
- サンプルデータで生成と検証ができる
- ユーザーが画面から必要情報を入力し、時間割候補を確認できる
- 違反と改善候補がユーザーに分かる
- 技術選定と実装範囲が過剰でない

## 完了条件
最終成果物は、ユーザー承認済み要件に基づいて実装、検証、改善され、ハード制約違反なしのサンプル時間割生成、評価ログ、操作可能な画面、入出力手段を備えた状態で完了とします。
