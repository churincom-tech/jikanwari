# User Input To Candidate Flow

## 目的
ユーザーが時間割作成に必要な情報を画面で入力し、その条件から時間割候補を表示する流れを定義します。

## 入力ステップ
1. 学校設定
   - 学年
   - クラス数
   - 曜日
   - 時限数
2. 授業設定
   - 教科ごとの週時数
   - 担当教員
   - 特別教室の有無
3. 固定・制約設定
   - 固定授業
   - 教員の勤務不可時間
   - 必ず守りたい条件
   - できれば守りたい条件

## 入力確認
候補生成前に、次を確認します。

- 必須項目が不足していないか
- 教科の必要時数が総時限数に収まるか
- 教員や特別教室の明らかな重複がないか
- 固定授業が同じクラス、同じ時限で重複していないか
- 勤務不可時間と固定授業が矛盾していないか

## 候補生成
初期試作では、まずハード制約を満たす候補を1案以上作ります。可能であれば、同じ入力条件から複数候補を作り、ソフト制約スコアで並べます。

## 候補表示
候補表示では、次を示します。

- クラス別時間割
- 教員別の重複有無
- 特別教室の重複有無
- ハード制約違反数
- ソフト制約スコア
- 改善候補

## 初期試作での優先順位
1. 画面入力から候補を生成して表示する
2. ハード制約違反を検出する
3. 候補ごとのスコアと警告を表示する
4. JSON保存またはエクスポートで入力条件と結果を残す

## 後続候補
- 手修正後の再検証
- 複数候補の詳細比較
- CSVインポート
- 印刷用表示
- PDF出力
