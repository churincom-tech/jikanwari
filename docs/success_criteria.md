# Success Criteria

## エージェント環境の成功条件
- `AGENTS.md` から必要文書へ迷わず進める
- 要件、仮定、未確認事項が分離されている
- Skillごとの責務と発動条件が明確である
- ループ、品質ゲート、既知失敗管理がある
- 認識確認結果が `artifacts/verification/recognition_check.md` に記録されている

## アプリ試作の成功条件
- 画面から学年、クラス数、曜日、時限数を入力できる
- 画面から教科ごとの週時数、担当教員、特別教室の有無を入力できる
- 画面から固定授業、勤務不可時間、必ず守りたい条件を入力できる
- ハード制約違反なしの時間割案を生成できる
- 候補時間割を少なくとも1案表示できる
- 可能なら複数候補をスコアや警告付きで比較できる
- 違反、偏り、空き時間、教室競合を検出できる
- 画面で生成結果を確認できる
- JSON保存またはエクスポートで生成条件と結果を再確認できる

## 最終成果物の成功条件
- 実用に近い学校条件で検証されている
- 制約違反と改善候補が説明可能である
- ユーザーが手修正できる
- 評価ログと改善履歴が残っている
- ユーザー承認済みの完了条件を満たしている
