# Constraints

## プロジェクト制約
- すぐにWebアプリ本体の実装へ入らない
- 既存プロジェクトがあれば技術と構成を優先する
- 既存プロジェクトがない場合は、後続工程で軽量な試作構成を比較する
- 完全無料のローカルWebアプリとして開発する
- 有料APIを使用しない
- 外部APIや外部CDNに依存しない初期試作にする
- 不明点は `context/assumptions.md` に分離する
- 検証データ、評価基準、改善ログなしに完成扱いしない

## セキュリティとプライバシー
- 教員名、生徒情報、学校内部情報はユーザー承認なしに外部送信しない
- 秘密情報や認証情報を無断利用しない
- nested Codexや外部サービス連携を無承認で行わない
- 初期試作では、生成処理と保存処理をユーザーのPC上で完結させる

## 時間割生成制約
- ハード制約違反を完成扱いしない
- ハード制約とソフト制約を混同しない
- アルゴリズムを初期段階から過剰に複雑化しない
- サンプルデータなしに生成品質を判断しない
