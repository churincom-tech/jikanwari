# Timetable Constraint Catalog

## ハード制約
| ID | 制約 | 検出方法 | 失敗時の扱い |
| --- | --- | --- | --- |
| H-001 | 同じ教員が同じ時限に複数クラスを担当しない | teacher_id と slot の重複確認 | 生成失敗または再配置 |
| H-002 | 同じ特別教室が同じ時限に複数授業へ割り当てられない | room_id と slot の重複確認 | 生成失敗または再配置 |
| H-003 | 各クラスの各教科が必要時数を満たす | LessonRequirement と結果数の比較 | 不足または超過を修正 |
| H-004 | 教員の勤務不可時間には授業を入れない | unavailable_slots と割当の照合 | 割当を禁止 |
| H-005 | 固定授業枠を変更しない | FixedAssignment の保持確認 | 固定枠を復元 |
| H-006 | クラスに同時刻で複数授業を入れない | class_id と slot の重複確認 | 生成失敗または再配置 |
| H-007 | 2時間連続必須の授業は連続する2コマで配置する | LessonRequirement.double_mode が required の授業について、同日隣接2時限のペアを検出 | 生成失敗または再配置 |
| H-008 | クラスの途中時限に空きを作らない | クラス・曜日ごとに、最後の授業時限より前の空き時限を検出 | 空き時限をその日の末尾へ寄せる |
| H-009 | 固定授業は授業情報の週時数を超えて置かない | LessonRequirement ごとの FixedAssignment 件数と weekly_count を比較 | 超過固定を禁止し、入力エラーにする |
| H-010 | 日ごとの授業数を通常の日課として自然な範囲に収める | クラスの週時数を曜日数で分配し、各曜日の使用時限上限を超える配置を検出 | 極端な短日課や過密日を避け、通常は 5-6 コマ程度に均す |
| H-011 | 2時間続きの授業は給食をまたがない | 同じ授業が4限と5限に連続していないかを検出 | 4-5限の連続配置を禁止し、1-2、2-3、3-4、5-6などへ移動 |
| H-012 | 2時間続き設定のない授業を連続させない | double_mode が none の授業が同日に隣接していないかを検出 | 理科・保体など通常授業の連続配置を避ける |

## ソフト制約
| ID | 制約 | 評価例 | 改善方針 |
| --- | --- | --- | --- |
| S-001 | 主要教科が特定曜日や午後に偏りすぎない | 曜日別、午前午後別の分散 | スコアで偏りを下げる |
| S-002 | 同じ教科が同じ日に重なりすぎない | class_id, subject_id, day の回数 | 別日に移動 |
| S-003 | 教員の空き時間や連続授業を平準化する | 教員別の空き数、連続数 | 教員負荷スコアを調整 |
| S-004 | 特別教室利用を分散する | room_type別の集中度 | 利用枠を入れ替える |
| S-005 | 非常勤教員の出勤日を少なくする | teacher_id別の出勤日数 | 同日内へ集約 |
| S-006 | クラスごとの日々の負荷を均等にする | 日別授業負荷スコア | 重い日から軽い日へ移動 |
| S-007 | 2時間連続が望ましい授業をなるべく連続させる | LessonRequirement.double_mode が preferred の授業について、作れるペア数と実際のペア数を比較 | 連続する2コマへ寄せる |

## ルール
ハード制約は必須条件です。ソフト制約はスコアリング対象であり、違反というより改善対象として扱います。
