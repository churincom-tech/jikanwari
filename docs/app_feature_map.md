# App Feature Map

## 初期画面候補
| 画面 | 目的 | 初期試作で扱うか |
| --- | --- | --- |
| 学校設定 | 曜日、時限、学年、クラスを設定 | 扱う |
| 教員設定 | 担当、勤務不可、非常勤条件を設定 | 扱う |
| 教科設定 | 週時数、教室条件、連続可否を設定 | 扱う |
| 教室設定 | 特別教室と利用可能枠を設定 | 扱う |
| 固定枠設定 | 学活、道徳、会議、行事を固定 | 扱う |
| 条件確認画面 | 入力不足、矛盾、必須条件を確認 | 扱う |
| 候補生成画面 | 入力条件から複数の時間割候補を生成 | 扱う |
| 候補表示画面 | 候補ごとの時間割、違反、スコアを比較 | 扱う |
| 検証画面 | 違反、偏り、スコア、改善候補を表示 | 扱う |
| 手修正画面 | 生成結果を調整 | 後続または簡易対応 |
| 入出力画面 | JSON保存、CSV/JSONの読み書き | 後続または簡易対応 |
| 印刷画面 | 印刷用表示、PDF出力 | 後続候補 |

## 初期ナビゲーション案
設定 -> 条件確認 -> 候補生成 -> 候補表示 -> 検証 -> 修正 -> 出力

## 初期入力フロー
1. 学校設定: 学年、クラス数、曜日、時限数
2. 授業設定: 教科ごとの週時数、担当教員、特別教室の有無
3. 制約設定: 固定授業、勤務不可時間、必ず守りたい条件
4. 条件確認: 不足、矛盾、ハード制約の可能性を確認
5. 候補生成: 複数案を作成し、スコア順に表示
