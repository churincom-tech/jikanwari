# Evaluation Plan

## 評価対象
- 入力データの妥当性
- ハード制約違反の有無
- ソフト制約スコア
- 教員負荷
- クラスごとの偏り
- 特別教室競合
- 2時間連続必須または望ましい授業の配置状態
- 手修正しやすさ
- 入出力の扱いやすさ

## 初期評価手順
1. 画面またはJSONから学校設定、授業設定、固定・制約設定を入力する
2. 候補生成前に入力不足と矛盾を確認する
3. 時間割候補を生成する
4. ハード制約違反を検出する
5. ソフト制約スコアを計算する
6. 2時間連続必須授業が隣接2コマで配置されているか確認する
7. 候補表示画面で違反、スコア、改善候補を確認する
8. 必要に応じて制約や生成手順を修正する

## 合格基準
- ハード制約違反が0件
- 必要時数の不足と超過が0件
- 2時間連続必須の授業がすべて連続2コマで配置されている
- 競合が一覧で確認できる
- ソフト制約の悪化理由が説明できる
- 再現可能なサンプルデータが残っている
- 入力条件と候補結果をJSONで保存またはエクスポートできる
