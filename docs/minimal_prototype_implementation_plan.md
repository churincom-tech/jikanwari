# Minimal Prototype Implementation Plan

## 目的
完全無料のローカルWebアプリとして、入力フォームから時間割候補を生成・表示する最小試作を実装するための計画です。

## 実装方針
- `docs/technology_stack_decision.md` の決定に従う
- 有料API、外部API、外部CDNを使わない
- 初期試作は `app/index.html` をブラウザで直接開ける構成にする
- すべての生成処理と検証処理をローカルJavaScriptで行う
- 実データ利用時も外部送信しない

## 初回実装のスコープ
| 区分 | 含める | 後続 |
| --- | --- | --- |
| 学校設定入力 | 学年、クラス数、曜日、時限数 | 短縮日課 |
| 授業設定入力 | 教科、週時数、担当教員、特別教室 | 学年別の細かい例外 |
| 固定・制約入力 | 固定授業、勤務不可時間、標準ハード制約 | 複雑な会議、行事 |
| 候補生成 | 最大3案、ハード制約優先 | 高度な最適化 |
| 候補表示 | クラス別時間割、違反、スコア | 教員別/教室別の高度表示 |
| 保存 | JSONエクスポート/インポート | CSV完全互換、PDF |
| 手修正 | 初期は対象外または簡易 | ドラッグ編集、履歴 |

## 実装フェーズ

### Phase 1: 静的アプリ土台
作成予定:
- `app/index.html`
- `app/styles.css`
- `app/scripts/main.js`
- `app/README.md`

完了条件:
- ブラウザで `app/index.html` を開ける
- 画面に学校設定、授業設定、固定・制約設定、候補表示エリアが見える
- 外部CDNや外部APIを参照していない

### Phase 2: データモデルとサンプルデータ
作成予定:
- `app/scripts/state.js`
- `app/scripts/sample-data.js`

実装内容:
- `GenerationRequest` 相当の状態を作る
- 3学年、各2クラス、週5日、6時限の匿名サンプルを用意する
- サンプルをフォームへ読み込めるようにする

完了条件:
- サンプル条件が画面に反映される
- 入力状態をJSONとして確認できる

### Phase 3: 入力フォーム
実装内容:
- 学校設定フォーム
- 授業設定テーブル
- 教員追加フォーム
- 教室追加フォーム
- 固定授業入力
- 勤務不可時間の曜日・時限グリッド

完了条件:
- `docs/minimal_prototype_screen_design.md` の必須項目を入力できる
- 入力値が内部状態に反映される

### Phase 4: 条件確認とハード制約検証
作成予定:
- `app/scripts/validation.js`

実装内容:
- 必須項目チェック
- 必要授業数が利用可能時限を超えないか確認
- 固定授業の重複確認
- 教員の勤務不可時間と固定授業の矛盾確認
- 教員重複、教室重複、クラス重複の検証関数

完了条件:
- 候補生成前のエラーと警告を表示できる
- 明らかな矛盾がある場合、生成ボタンを止められる

### Phase 5: 候補生成
作成予定:
- `app/scripts/timetable-core.js`
- `app/scripts/scoring.js`

実装内容:
- 固定授業を先に配置する
- 必要授業を未配置リストへ展開する
- 制約が強い授業からローカル探索で配置する
- 探索順を変えて最大3候補を作る
- ハード制約違反が残る候補を完成扱いしない
- ソフト制約スコアを計算する

完了条件:
- サンプルデータで少なくとも1候補を生成できる
- 候補ごとにハード制約違反数とソフト制約スコアがある

### Phase 6: 候補表示
実装内容:
- 候補比較エリア
- クラス別時間割グリッド
- 候補詳細パネル
- セルの固定、警告、未配置、競合表示

完了条件:
- 候補A/B/Cを比較できる
- 選択中候補のクラス別時間割を確認できる
- 違反、スコア、改善候補が見える

### Phase 7: JSON保存と読み込み
作成予定:
- `app/scripts/import-export.js`

実装内容:
- 入力条件と生成結果をJSONでダウンロード
- JSONファイルを読み込んで復元
- localStorageで直近状態を補助保存

完了条件:
- 入力条件と候補結果をJSONで保存できる
- 保存したJSONを読み直して再確認できる

### Phase 8: 検証
実装内容:
- 手動確認チェックリストを実行
- サンプルデータで生成できることを確認
- ハード制約違反が0件の候補を確認
- 外部通信参照がないことを確認

完了条件:
- `artifacts/verification/` に検証結果を記録する
- 既知の失敗や残課題を `loops/improvement_backlog.md` に反映する

## 受け入れ条件
- 完全無料で動く
- ローカルブラウザで動く
- 有料API、外部API、外部CDNを使っていない
- 学年、クラス数、曜日、時限数を入力できる
- 教科ごとの週時数、担当教員、特別教室の有無を入力できる
- 固定授業、勤務不可時間、必ず守りたい条件を入力できる
- 入力不足や矛盾を候補生成前に表示できる
- ハード制約違反なしの候補を少なくとも1案表示できる
- 候補ごとのスコア、警告、改善候補を表示できる
- 入力条件と候補結果をJSONで保存できる

## 実装しないこと
- 有料APIの利用
- 外部APIの利用
- 外部CDNの利用
- クラウド保存
- 認証
- 複数校管理
- 高度な権限管理
- PDF出力
- Excel完全互換
- AIによる自動判断

## 実装開始前の承認事項

`/goal 入力フォームから時間割候補を生成表示する無料ローカルWebアプリの最小試作を実装する`

承認時の前提:
- 技術スタックはHTML/CSS/Vanilla JavaScript
- 有料API、外部API、外部CDNなし
- 初期候補は最大3案
- サンプル規模は3学年、各2クラス、週5日、6時限
- JSON保存を初期対応する
