# Minimal Prototype Screen Design

## 目的
ユーザーが時間割作成に必要な情報を画面で入力し、その条件から時間割候補を生成して確認できる最小試作の画面設計を定義します。

この文書は実装前の設計です。ここで扱うのは、入力フォーム項目、候補表示画面、入力検証、生成結果の見せ方、実装前確認事項です。

## 最小試作の前提
| 項目 | 初期設計 |
| --- | --- |
| 対象 | 1週間分の中学校時間割 |
| 入力方法 | 画面手入力を主、JSON保存を補助 |
| 候補数 | 初期は最大3案。生成できない場合は理由を表示 |
| 最低合格 | ハード制約違反なしの候補を1案以上表示 |
| 手修正 | 初期は候補確認中心。直接編集は後続または簡易対応 |
| CSV | 後続または簡易インポート候補 |
| PDF/印刷 | 後続候補 |

## 画面構成
| 順 | 画面 | 目的 | 最小試作で必要な状態 |
| --- | --- | --- | --- |
| 1 | 学校設定 | 学年、クラス、曜日、時限を入力 | 必須 |
| 2 | 授業設定 | 教科、週時数、担当教員、教室条件を入力 | 必須 |
| 3 | 固定・制約設定 | 固定授業、勤務不可、必須条件を入力 | 必須 |
| 4 | 条件確認 | 入力不足と矛盾を確認 | 必須 |
| 5 | 候補生成 | 入力条件から時間割候補を作る | 必須 |
| 6 | 候補表示 | 候補の時間割、違反、スコアを比較 | 必須 |
| 7 | JSON保存 | 入力条件と生成結果を保存 | 簡易対応 |

## 画面2: 授業設定
教科ごとの必要時数と担当者を、表形式で入力します。初期試作では、学年・クラス別の細かい例外よりも「クラスごとの教科必要時数」を優先します。

| ID | ラベル | データ項目 | 入力UI | 必須 | 初期値 | 検証 |
| --- | --- | --- | --- | --- | --- | --- |
| subject.name | 教科名 | `subjects[].name` | テキストまたは候補選択 | 必須 | 国語など | 重複名に警告 |
| lesson.class | 対象クラス | `lesson_requirements[].class_id` | クラス選択 | 必須 | 全クラス行を生成 | 存在するクラス |
| lesson.weeklyCount | 週時数 | `lesson_requirements[].weekly_count` | 数値入力 | 必須 | 0 | 0から総時限数 |
| lesson.teacher | 担当教員 | `lesson_requirements[].teacher_id` | 教員選択または追加 | 必須 | 未設定 | 未設定は生成不可 |
| subject.roomType | 特別教室 | `subjects[].room_type` | 選択 | 任意 | 普通教室 | 必要教室が未登録なら警告 |
| subject.allowDouble | 連続授業可 | `subjects[].rules.allow_double` | トグル | 任意 | 不可 | ソフト制約に反映 |
| subject.sameDayLimit | 1日上限 | `subjects[].rules.same_day_limit` | 数値入力 | 任意 | 1 | 1以上 |

| teacher.partTime | 非常勤 | `teachers[].part_time` | トグル | 任意 | false | ソフト制約に反映 |

| room.type | 教室種別 | `rooms[].room_type` | 選択 | 必須 | 普通教室 | 教科の必要種別と照合 |

## 画面3: 固定・制約設定
| ID | ラベル | データ項目 | 入力UI | 必須 | 初期値 | 検証 |
| --- | --- | --- | --- | --- | --- | --- |
| fixed.class | 固定するクラス | `fixed_assignments[].class_id` | クラス選択 | 任意 | 空 | 存在するクラス |
| fixed.slot | 曜日・時限 | `fixed_assignments[].slot` | グリッド選択 | 任意 | 空 | 同一クラス同時刻の重複不可 |
| fixed.subject | 教科 | `fixed_assignments[].subject_id` | 教科選択 | 任意 | 空 | 教科の必要時数に含める |
| fixed.teacher | 教員 | `fixed_assignments[].teacher_id` | 教員選択 | 任意 | 空 | 勤務不可時間と矛盾ならエラー |
| fixed.room | 教室 | `fixed_assignments[].room_id` | 教室選択 | 任意 | 普通教室 | 同一教室同時刻の重複不可 |
| hardRules.required | 必ず守りたい条件 | `constraints.hard` | チェックボックス | 必須 | 標準ハード制約 | 標準ハード制約は解除不可 |
| softRules.preferred | できれば守りたい条件 | `constraints.soft` | チェックボックス | 任意 | 標準ソフト制約 | スコアに反映 |

### 標準ハード制約
- 同じ教員が同じ時限に複数クラスを担当しない
- 同じ特別教室が同じ時限に複数授業へ割り当てられない
- 各クラスの各教科が必要時数を満たす
- 教員の勤務不可時間には授業を入れない
- 固定授業枠を変更しない
- クラスに同時刻で複数授業を入れない

### 標準ソフト制約
- 主要教科が特定曜日や午後に偏りすぎない
- 同じ教科が同じ日に重なりすぎない
- 教員の空き時間や連続授業をなるべく平準化する
- 特別教室利用をなるべく分散する
- 非常勤教員の出勤日をなるべく少なくする
- クラスごとの日々の負荷をなるべく均等にする

## 画面4: 条件確認
候補生成前に、入力値から生成できる状態かを確認します。

| 種別 | 表示内容 | 例 | 次操作 |
| --- | --- | --- | --- |
| エラー | 生成前に直す必要がある問題 | 1年1組の国語担当教員が未設定 | 該当入力へ移動 |
| エラー | ハード制約が最初から矛盾している | 月曜1限に同じ教室の固定授業が2件 | 該当固定枠へ移動 |
| 警告 | 生成はできるが品質に影響する問題 | 体育館が1つで体育が同じ曜日に集中 | そのまま生成可 |
| 情報 | 入力規模と総授業数 | 週30時限、必要授業数29 | 確認のみ |

### 条件確認の合格条件
- 必須項目の未入力がない
- 各クラスの必要授業数が利用可能時限を超えない
- 固定授業同士の明らかな重複がない
- 勤務不可時間と固定授業の矛盾がない

## 画面5: 候補生成
| 項目 | 初期設計 |
| --- | --- |
| 生成ボタン | 条件確認が合格または警告のみのとき有効 |
| 候補数 | 最大3案 |
| 生成中表示 | 進行中、試行回数、現在の最良スコア |
| 失敗時表示 | 生成できなかった理由、修正候補、問題のある制約 |
| 再生成 | 同じ条件で候補を作り直す |
| 部分固定 | 後続候補。初期は固定授業のみ対応 |

## 画面6: 候補表示
候補表示は、上部に候補比較、下部に選択中候補の時間割グリッドを置きます。

### 候補比較エリア
| 表示項目 | 内容 | 目的 |
| --- | --- | --- |
| 候補名 | 候補A、候補B、候補C | 比較しやすくする |
| 総合スコア | 100点満点または低ペナルティ順 | 良さをざっくり判断 |
| ハード制約違反 | 件数と重大度 | 0件でない候補を完成扱いしない |
| ソフト警告 | 偏り、連続授業、空き時間など | 改善余地を把握 |
| 教員負荷 | 空き時間、連続授業の目安 | 実用性を確認 |
| 教室競合 | 特別教室の重複有無 | 教室制約を確認 |
| 操作 | 詳細表示、JSON保存、再生成 | 次操作につなげる |

### 時間割グリッド
| 表示項目 | 内容 |
| --- | --- |
| 表示切替 | クラス別を初期表示。教員別、教室別は後続または簡易対応 |
| 行 | 時限 |
| 列 | 曜日 |
| セル内容 | 教科、担当教員、教室 |
| セル状態 | 固定、警告、未配置、競合をバッジ表示 |
| 詳細 | セル選択で割当理由、関連制約、警告を表示 |

### 候補詳細パネル
| セクション | 表示内容 |
| --- | --- |
| ハード制約 | 違反0件なら合格表示。違反があれば制約ID、対象、修正候補 |
| ソフト制約 | スコア内訳、偏り、空き時間、連続授業 |
| 未配置 | 生成できなかった授業があれば教科、クラス、理由 |
| 改善候補 | 例: 体育を火曜3限から木曜2限へ移すと偏りが下がる |

## 画面内コピー方針
| 状態 | 表示文 |
| --- | --- |
| 入力不足 | 必須項目が残っています。赤い項目を入力すると候補生成できます。 |
| 警告あり | 候補生成はできますが、品質に影響する条件があります。 |
| 生成成功 | ハード制約違反なしの候補を生成しました。 |
| 生成失敗 | この条件では候補を作れませんでした。固定授業または勤務不可時間を見直してください。 |
| 候補比較 | スコアと警告を見比べて、採用候補を選んでください。 |

## データ構造への対応
| 画面入力 | データモデル |
| --- | --- |
| 学校設定 | `School`, `Grade`, `ClassGroup`, `Slot` |
| 授業設定 | `Subject`, `Teacher`, `LessonRequirement`, `Room` |
| 固定・制約設定 | `FixedAssignment`, `constraints` |
| 生成条件 | `GenerationRequest` |
| 候補 | `TimetableCandidate` |
| 検証結果 | `ValidationResult` |

## 最小試作の受け入れ条件
- 学年、クラス数、曜日、時限数を画面で入力できる
- 教科ごとの週時数、担当教員、特別教室の有無を画面で入力できる
- 固定授業、勤務不可時間、必ず守りたい条件を画面で入力できる
- 候補生成前に不足と矛盾を表示できる
- 時間割候補を少なくとも1案表示できる
- 候補ごとにハード制約違反数とソフト制約スコアを表示できる
- ハード制約違反がある候補を完成扱いしない
- 入力条件と候補結果をJSONで保存またはエクスポートできる

## 実装前確認事項
| 確認事項 | 初期推奨 | 理由 |
| --- | --- | --- |
| 最初の候補表示数 | 最大3案 | 比較できるが画面が重くなりにくい |
| 試作用の学校規模 | 3学年、各2クラス、週5日、6時限 | 中学校らしさと軽さのバランス |
| 入力方法 | 手入力中心、JSON保存付き | 開発経験がないユーザーでも試せる |
| 手修正 | 初期は詳細設計のみ | まず候補生成と検証を安定させる |
| CSV | 後続 | 最初は入力画面の理解しやすさを優先 |

## 後続改善候補
- 候補を採用して一部固定する
- セルをドラッグして手修正する
- 手修正後に再検証する
- CSVインポートとエクスポート
- 教員別、教室別の印刷表示
- PDF出力
