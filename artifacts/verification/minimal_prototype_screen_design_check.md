- `docs/minimal_prototype_screen_design.md`

- `context/assumptions.md`

## 要件別確認
| 要件 | 証跡 | 判定 |
| --- | --- | --- |
| 入力フォーム項目を定義している | `docs/minimal_prototype_screen_design.md` の学校設定、授業設定、固定・制約設定 | 合格 |
| 学年、クラス数、曜日、時限数を扱う | 学校設定の項目表 | 合格 |
| 教科ごとの週時数、担当教員、特別教室の有無を扱う | 授業設定の項目表 | 合格 |
| 固定授業、勤務不可時間、必ず守りたい条件を扱う | 固定・制約設定の項目表 | 合格 |
| 候補表示画面を設計している | 候補比較エリア、時間割グリッド、候補詳細パネル | 合格 |
| 候補ごとの違反、スコア、改善候補を表示する | 候補比較エリアと候補詳細パネル | 合格 |
| ハード制約違反を完成扱いしない | 候補比較エリア、受け入れ条件 | 合格 |
| 実装前確認事項を分離している | 実装前確認事項、`context/assumptions.md` | 合格 |
| アプリ本体実装に入っていない | 文書追加と既存文書更新のみ | 合格 |

## 品質ゲート確認
| 観点 | 判定 | メモ |
| --- | --- | --- |
| 目的適合 | 合格 | ユーザー入力から候補表示までに集中 |
| 対象者適合 | 合格 | 学校現場向けのラベルと表示文を使用 |
| 制約遵守 | 合格 | 実装開始、外部送信、重要方針変更なし |
| 過剰設計なし | 合格 | 手修正、CSV、PDFは後続候補へ分離 |
| 制約分離 | 合格 | 標準ハード制約とソフト制約を分離 |
| 検証可能性 | 合格 | 受け入れ条件と評価計画への接続あり |
| UI分かりやすさ | 合格 | 入力不足、警告、生成失敗、候補比較の表示文を定義 |

## 結論
入力フォーム項目と時間割候補表示画面の最小試作設計は、現時点の要件に対して完了しています。次工程は、ユーザー承認後の技術スタック選定または最小試作実装です。
